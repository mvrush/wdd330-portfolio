Notes on Objects

An object in JavaScript is a self-contained set of related values and functions. They act as a collection of named properties that map to any JavaScript value such as stings, numbers, booleans, arrays and even functions. If a property's value is a function, it is known as a method.
This is an example of a JavaScript Object which has all the different JavaScript values I listed earlier in it. It looks very similar to JSON format:
const superman = {
name: 'Superman',
'real name': 'Clark Kent',
height: 75,
weight: 235,
hero: true,
villian: false,
allies: ['Batman','Supergirl','Superbaby'],
fly() {
return 'Up, up and away!';
  }
};


Notes on the DOM

When you see a legacy DOM shortcut method like Document.body it is adressing everything found in the HTML Document under the <body> tag.
The same thing is true when you see something like Document.links. It is addressing everything found in the HTML Document that has an <a> or <area> tag and an href attribute.

One thing to remember is that when using CSS property names that are separated by dashes such as 'background-color' it must be written in camelCase notation in JavaScript. So it becomes 'backgroundColor'. Here's an example of it in use in JavaScript.
superman.style.backgroundColor = 'blue';
Best practice is to add different CSS classes on the CSS stylesheet and then toggle them with JavaScript rather than writing CSS elements on the fly with JavaScript.


Notes on Events

Events are basically user interactions with the webpage. They can be by mouse click, scroll wheel, keyboard press or release. There are also touch events that can be used with touchscreens to trigger things.
These events can be used to trigger JavaScript.
