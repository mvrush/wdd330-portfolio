WDD 330 Week 02

Notes:

One important thing I learned this week was the use of Template Literals by using backticks. It allows you to use both single and double quotes in a string. But the real power is that they let you interpolate JavaScript code inside the string. That means you can insert a JavaScript expression. For example it allows you to insert variable values into strings.
So instead of concatenating with + like this:
"hello" + name
You can simply write:
`Hello ${name}`
and it will output the string with the name contained in the variable.


Questions:

One question I had from the reading is this.
Is there a preference to use Function Declarations like this:
function hello() {
console.log('Hello World!'); }
Or to use Function Expressions which store the function in a variable. Like this:
const goodbye = function () {
console.log('Goodbye World!'); }
Also, is there a time where you would use one instead of the other?
