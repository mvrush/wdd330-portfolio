Notes:

This week from the reading I was able to pick up some new testing and debugging methods. One of the more useful methods was to employ the alert() to effectively stop a program from executing at some point until the alert is clicked. It's a handy way to stop program execution.

I also got a lot of information on Error Objects and how to create them. We use error objects with a constructor function to create custom error objects. For example we can use the TypeError() to show when a string is used but a number was expected. Here's the example:
const error = new TypeError('You need to use numbers in this function');
When a type mismatch is detected the error will be thrown.

I also went more in-depth on the browser console's built-in debugging tool. I found it very handy to insert breakpoints into my code to check it's functionality while running.
